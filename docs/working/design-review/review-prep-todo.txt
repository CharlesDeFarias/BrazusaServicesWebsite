Design Review Prep TODO

Goal:
Prepare the most useful possible Claude design review packet without gathering unnecessary extra material.

1. Write a short observation memo
- Create a concise note covering:
- what feels wrong
- what feels strong
- what feels uncertain
- Keep it short and specific. Aim for the issues that are most noticeable in real use, especially on mobile.

2. Identify the highest-priority concerns
- Pick the top 3 concerns you most want the review to focus on.
- Good examples:
- hero still feels too tall or dense on mobile
- awkward spacing or blank gaps between sections
- inconsistent section rhythm or transitions
- image choice undermining trust or polish

3. Mark what already feels strong
- List 2 things that already feel good and should not be over-corrected.
- Good examples:
- typography direction
- navy/gold system
- section structure
- overall tone

4. Separate issue types
- For each major concern, label it as one of:
- layout bug
- visual design issue
- copy-density issue
- not sure
- This will help Claude avoid mixing implementation bugs with broader design judgment.

5. Create a screenshot/video priority list
- Make a short list of which files matter most.
- Mark them as high / medium / low priority.
- Especially highlight:
- the strongest examples
- the weakest examples
- the clearest bug/spacing captures

6. Pull out the most relevant market-research guidance
- From `docs/briefs/marketresearch.txt`, note:
- 2-3 points that should actively shape the review
- 1-2 patterns you like
- 1-2 patterns you do not want
- Do not make Claude infer everything from the full file if you already know what matters most.

7. List locked constraints
- Write down what is already decided and should not be reopened casually.
- Likely items:
- single-page `/clean` stays for now
- navy/gold direction stays
- current hero CTA direction is locked
- accordion headline is locked
- copy-layer architecture is already in place

8. Define the review scope
- Decide what kind of review you want.
- Recommended scope:
- not a full redesign
- not implementation yet
- review mobile hierarchy
- review section rhythm and spacing
- review transitions between sections
- review whether current visuals support the current positioning
- review whether any image choices weaken trust

9. Write 2 specific questions you want answered
- Examples:
- What is the single biggest visual problem on mobile right now?
- Which section transition feels weakest and why?
- Does the current visual structure support the stronger operator-focused positioning?

10. Optional: annotate only the most important captures
- If helpful, annotate only a few files.
- Use arrows or short notes like:
- too much blank space
- feels dense
- transition feels abrupt
- image feels generic
- Do not annotate everything unless it clearly adds value.

11. Build the final Claude review packet
- Include:
- `docs/briefs/marketresearch.txt`
- `docs/decisions.md`
- `docs/working/design-review/desktop/`
- `docs/working/design-review/mobile/`
- your short observation memo
- your screenshot/video priority list
- your locked constraints
- your 2 review questions

12. Keep the prompt narrow
- Ask Claude for:
- findings first
- ranked by impact
- clear separation between layout bugs, design issues, and copy-density issues
- no implementation yet unless requested later
